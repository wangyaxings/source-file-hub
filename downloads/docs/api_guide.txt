FileServer API 使用指南
========================

这是一个安全的HTTPS REST API服务器，支持用户认证和文件下载。

## 基础信息

- 服务地址: https://localhost:8443
- 协议: HTTPS (TLS加密)
- 认证方式: Bearer Token

## 认证流程

1. 获取默认用户信息
   GET /api/v1/auth/users

2. 用户登录获取token
   POST /api/v1/auth/login
   Body: {"tenant_id": "demo", "username": "admin", "password": "admin123"}

3. 使用token访问受保护资源
   Headers: Authorization: Bearer <your_token>

## 文件下载

所有文件下载都需要认证，统一通过以下接口：

GET /api/v1/files/{文件路径}

支持的文件类型：
- 配置文件: configs/config.json
- SSL证书: certificates/server.crt, certificates/server.key
- 证书信息: certificates/cert_info.json
- 文档: docs/api_guide.txt

## 示例

curl -k -H "Authorization: Bearer <token>" \
  https://localhost:8443/api/v1/files/configs/config.json

curl -k -H "Authorization: Bearer <token>" \
  https://localhost:8443/api/v1/files/certificates/server.crt

## 安全特性

- 所有通信使用HTTPS加密
- 路径遍历攻击防护
- 文件路径白名单验证
- Token过期自动清理
- 详细的访问日志记录
